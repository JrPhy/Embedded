STM32F429 DHT22 溫濕度監測專案
本專案實作了在 STM32F429I-Discovery 開發板上驅動 DHT22 (AM2302) 感測器的功能。透過精確的微秒級時序控制與外部中斷 (EXTI) 擷取技術，讀取環境中的溫度與濕度，並將結果顯示於開發板內建的 LCD 螢幕上。

## 一、功能特點
1. 高精度時序控制：利用 TIM1 計時器實作微秒 (us) 級延遲，確保符合 DHT22 的單線式 (One-Wire) 通訊協定。
2. 中斷驅動解析：在 dht22.c 中透過 GPIO 外部中斷 (HAL_GPIO_EXTI_Callback) 記錄脈衝寬度，以判斷邏輯 0 或 1，提高系統即時性。
3. 自動數據校驗：支援 40-bit 原始數據解析，包含濕度、溫度以及 Checksum 校驗碼驗證，確保數據準確性。
4. 即時顯示界面：整合 STM32F429 內建的 LTDC LCD 驅動，即時更新環境溫濕度數值。

## 二、硬體連接
1. 開發板：STM32F429I-DISCO
2. 感測器：DHT22 (AM2302)
3. 引腳配置：
Data Pin：連接至開發板的 GPIOE Pin 2 (PE2)。
VCC：接 3.3V 或 5V (視模組規範而定)。
GND：接開發板地線。

## 四、快速上手
1. 環境準備：使用 STM32CubeIDE 或 Keil MDK 開啟專案。.ioc 定義了時鐘樹 (180MHz) 與引腳功能。
2. 硬體接線：將 DHT22 的數據引腳連接至 PE2。
3. 編譯與燒錄：編譯專案並透過 ST-Link 燒錄至開發板。

## 五、觀察結果：
1. 開機後，LCD 螢幕將初始化並顯示專案標題。
2. 系統每隔一段時間會自動讀取數據，並在螢幕上更新 Temperature 與 Humidity 數值。

## 六、注意事項
1. 時序敏感：DHT22 對時序要求極高，若讀取失敗請確認 TIM1 的 Prescaler 設定（專案中預設為 71，對應 72MHz 頻率以實現 1us 計數）。
2. 拉高電阻：若使用裸件感測器而非模組，請務必在 Data Pin 與 VCC 之間接一個 4.7k~10k Ω 的上拉電阻。
